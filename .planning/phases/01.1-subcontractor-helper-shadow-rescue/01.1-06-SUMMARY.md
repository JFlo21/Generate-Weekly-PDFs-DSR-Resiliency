# Phase 01.1 Plan 06: SUB-09 Helper Partition Gap-Closure Summary

One-liner: Gate subcontractor legacy helper key emission on `not is_subcontractor_row`, fix 6-part-key hash-prune parser, extend TARGET cleanup with variant whitelist and shared scope builder.

## Objective

UAT run 26138204743 confirmed WR_90773033 wk 041226 Chris_Lopez produced both a legacy `_Helper_Chris_Lopez.xlsx` on TARGET_SHEET_ID and the correct `_ReducedSub_Helper_Chris_Lopez.xlsx` on PPP — dual upload, duplicate billing risk. Root cause: Bug-B1 (Plan 01.1-02) applied `not is_subcontractor_row` to the primary emission path but not the legacy helper emission path. This plan closes that asymmetry.

## What Was Built

### Task 1 — Helper-path partition gate (2805a63)

In `group_source_rows`, the legacy helper block at `keys_to_add.append(('helper', helper_key, helper_foreman))` was wrapped in `if not is_subcontractor_row:`. Subcontractor helper rows now emit only `reduced_sub_helper` / `aep_billable_helper` keys (partitioning contract). An `else:` DEBUG branch logs `"EXCLUDING from main Excel (subcontractor legacy helper): ..."` for operator visibility — body covered by the existing `"EXCLUDING from main Excel"` PII marker.

### Task 2 — Shared scope builder + TARGET cleanup extension (e8d1003)

New module-level `_build_subcontractor_wr_scope(groups) -> set[str]` extracts the WR numbers of all subcontractor variant groups (union of `reduced_sub`, `aep_billable`, `reduced_sub_helper`, `aep_billable_helper`). Used at the TARGET cleanup call site: when `SUBCONTRACTOR_LEGACY_HELPER_CLEANUP_ENABLED` is truthy, passes `sub_wr_scope=_sub_scope, sub_offcontract_variants={'helper', 'primary'}` to `cleanup_untracked_sheet_attachments`. Any TARGET attachment for a subcontractor WR with a `helper` or `primary` filename variant is unconditionally deleted, regardless of `valid_wr_weeks` or `KEEP_HISTORICAL_WEEKS`. The PPP cleanup call passes `None` to preserve existing whitelist-only behaviour.

`cleanup_untracked_sheet_attachments` signature extended with two keyword-only params defaulting to `None`:
- `sub_wr_scope: set[str] | None` — WR numbers subject to off-contract variant cleanup
- `sub_offcontract_variants: set[str] | None` — variant names that are off-contract for subcontractor WRs

### Task 3 — Hash-prune v2 + 6-part-key fix (e613e90)

`PHASE_1_1_HASH_PRUNE_VERSION` bumped `1` -> `2`. The v1 prune had `if len(_parts) != 4: continue` — a fatal exact-length guard that silently skipped all 6-part helper keys. v2 uses `if len(_parts) < 4: continue` and index-accesses `_parts[0]` (wr) / `_parts[2]` (variant) so 4-part, 5-part, and 6-part keys all parse. Orphan condition extended to `or _hk_variant == 'helper'` so subcontractor legacy helper hash entries are pruned in the same pass as primary orphans. Shared `_build_subcontractor_wr_scope(groups)` replaces the inline scope-build loop to prevent drift.

## Deviations from Plan

### Auto-fixed Issues

None — plan executed exactly as written. The 6-part-key hash-prune bug (`!= 4` guard silently skipping all helper keys) was identified in the planning phase and addressed by Task 3 as designed.

### Three-site invariant verification (documented per plan)

Sites 2 (`valid_wr_weeks`) and 3 (`current_keys`) required no explicit edits — both derive from `groups/__variant`. Because Task 1 removes subcontractor `'helper'` groups from the `groups` dict at the producer, neither site emits the orphan key. This self-healing behaviour was confirmed by the e2e tests in `TestEndToEndPipeline`.

## Decisions Made

1. `not is_subcontractor_row` applied to legacy helper block (mirrors Bug-B1 primary fix — symmetric approach)
2. `PHASE_1_1_HASH_PRUNE_VERSION` bumped to 2 (not a new constant) to reuse the existing version-gate mechanism
3. Minimum-length guard `< 4` chosen over exact-length `!= 4` to future-proof against any additional key-part variants
4. Shared `_build_subcontractor_wr_scope` extracted (prevents cleanup/prune scope drift)
5. `SUBCONTRACTOR_LEGACY_HELPER_CLEANUP_ENABLED` default ON (destructs only pre-existing duplicate attachments for subcontractor WRs; non-sub WR cleanup behaviour unchanged)
6. In-place test rewrites chosen over new test classes (per [2026-05-20 00:26] rule 2: preserve git-blame, add docstring, add sibling test for new invariant)
