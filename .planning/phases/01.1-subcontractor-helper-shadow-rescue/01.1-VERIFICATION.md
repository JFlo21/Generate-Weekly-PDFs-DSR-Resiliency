### Data-Flow Trace (Level 4)

This phase modifies routing/partitioning logic in a batch pipeline. The Plan
01.1-06 change is a boolean gate (not a new data path). The SUB-09 off-contract
gate in `cleanup_untracked_sheet_attachments` receives `sub_wr_scope` (a
`set[str]` of WR tokens built from real `groups` dict entries) and `valid_wr_weeks`
(a `set` of 4-tuples built from real `generated_filenames`). Both are populated
from real production data during the run — no hollow or hardcoded-empty data flows
exist in the new gate.

The `ident not in valid_wr_weeks` guard at L2772 compares the attachment's parsed
identity tuple against identities confirmed to have been generated (or validated
via hash-skip) this run. A cross-sheet-overlap live attachment has its identity
in `valid_wr_weeks` by construction (it was generated this run from a non-sub
sheet) → correctly exempted. A pre-fix orphan (never generated after Bug-B1
landed) has no identity in `valid_wr_weeks` → correctly deleted. Data flow is
correct and not hollow.
