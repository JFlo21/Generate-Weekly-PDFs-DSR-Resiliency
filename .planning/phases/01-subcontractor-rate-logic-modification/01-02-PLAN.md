<objective>
Extend the filename round-trip parser and the change-detection hash so the new subcontractor
variants flow through the existing variant-aware infrastructure without churning hashes for
unrelated variants.

Purpose: Plan 3 (variant generation) and Plan 4 (dual routing) both depend on (a) `build_group_identity`
correctly classifying the new filenames so attachment cleanup, hash-history lookups, and
target-row matching find the right files, and (b) `calculate_data_hash` invalidating cached
files when the CSV changes — but only for the variants that actually consume the CSV. Without
the fingerprint-scoping in D-20, every CSV edit would force regeneration of every primary /
helper / vac_crew file in the run.

Output: `build_group_identity` recognizes 4 new variants; `calculate_data_hash` injects
`_SUBCONTRACTOR_RATES_FINGERPRINT` into `meta_parts` for new variants only; the existing
source-side pre-scan and `_PII_LOG_MARKERS` list are confirmed (and extended where required)
to handle the new variant strings; comprehensive parser + hash regression tests.
</objective>
