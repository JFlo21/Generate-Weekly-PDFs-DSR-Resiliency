   INFO logs (`"REDUCED SUB HELPER GROUP CREATED"` and `"AEP BILLABLE HELPER GROUP CREATED"`).
   The current substring match (`"HELPER GROUP CREATED"`) catches them by accident — a future
   wording rewording would silently leak foreman names to Sentry. Per Living Ledger
   2026-04-20 12:00, explicit markers are the defense-in-depth contract.

2. **WR-06** — Standardize on `__source_sheet_id` in the missing-CU attribution loop inside
   `main()` (which currently reads `__sheet_id`). The two fields are aliased at populate
   time inside `_fetch_and_process_sheet`, so behavior is identical today, but the
   inconsistency is masked tech debt: a future refactor that splits the two field names
   would silently bucket missing-CU WARNINGs under sheet `-1` instead of the real sheet.

Bundle rationale: both fixes are pure tightening of existing safety nets — WR-04 hardens
the PII sanitizer's coverage of an already-scrubbed log line; WR-06 picks the canonical
field name where both currently exist. Neither touches production logic, both are
independent in code scope (PII markers tuple vs missing-CU attribution loop), and both
fit naturally into the same test file. Bundling minimizes plan count without adding
coupling.
